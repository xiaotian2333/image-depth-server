# 深度估计服务接口文档

## 基本信息

- 默认地址：`http://<host>:7860`
- 鉴权：无
- 数据格式：请求使用 URL Path 参数，响应为 `application/json`
- CORS：服务端当前允许跨域请求
- hash 规则：默认要求 20 位数字，可由服务端启动参数 `--hash-digits` 调整

客户端应把服务端返回的 `dataUrl` 当作完整图片地址使用，不需要再拼接 MIME 前缀。

---

## 1. 获取深度图

### 请求

```http
GET /depth/{hash}
```

### Path 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `hash` | string | 是 | 酷狗封面数字 hash，默认必须是 20 位数字 |

示例：

```http
GET /depth/20190917112902491378
```

### 成功响应

状态码：`200 OK`

```json
{
  "hash": "20190917112902491378",
  "width": 400,
  "height": 400,
  "dataUrl": "data:image/png;base64,iVBORw0KGgoAAA...",
  "createdAt": 1793376000000
}
```

### 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `hash` | string | 请求中的 hash |
| `width` | number | 返回深度图宽度，受服务端 `--max-image-size` 限制 |
| `height` | number | 返回深度图高度，受服务端 `--max-image-size` 限制 |
| `dataUrl` | string | PNG 灰度深度图的 Data URL，可直接赋给 `img.src` |
| `createdAt` | number | 生成时间，Unix 毫秒时间戳 |

### TypeScript 类型

```ts
export interface DepthResponse {
  hash: string;
  width: number;
  height: number;
  dataUrl: string;
  createdAt: number;
}
```

### 前端调用示例

```ts
const resp = await fetch("http://localhost:7860/depth/20190917112902491378");

if (!resp.ok) {
  const err = await resp.json();
  throw new Error(err.error ?? `HTTP ${resp.status}`);
}

const depth: DepthResponse = await resp.json();
imageElement.src = depth.dataUrl;
```

## 2. 健康检查

### 请求

```http
GET /health
```

### 成功响应

状态码：`200 OK`

```json
{
  "status": "ok",
  "cacheSize": 12,
  "modelInput": {
    "width": 518,
    "height": 518
  }
}
```

### 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | 服务状态，正常时为 `ok` |
| `cacheSize` | number | 当前 sled 缓存条目数量 |
| `modelInput.width` | number | ONNX 模型实际输入宽度 |
| `modelInput.height` | number | ONNX 模型实际输入高度 |

---

## 错误响应

错误响应统一为 JSON：

```json
{
  "error": "错误说明"
}
```

| 状态码 | 场景 | 客户端处理建议 |
| --- | --- | --- |
| `400 Bad Request` | hash 位数不符合服务端配置，或包含非数字字符 | 检查传入 hash |
| `413 Payload Too Large` | CDN 图片响应超过服务端大小限制 | 不建议重试，换 hash 或调整服务端配置 |
| `422 Unprocessable Entity` | 下载内容不是可解码图片 | 检查 hash 对应资源是否存在或有效 |
| `502 Bad Gateway` | 酷狗 CDN 下载失败、超时或返回非 2xx | 可稍后重试 |
| `500 Internal Server Error` | 模型加载、推理、缓存或 PNG 编码失败 | 记录错误并联系服务端排查 |

TypeScript 错误类型：

```ts
export interface ErrorResponse {
  error: string;
}
```

## 对接注意事项

- `GET /depth/{hash}` 首次请求会下载图片并执行模型推理，耗时明显高于缓存命中。
- 相同 hash 在同一模型版本下会命中服务端缓存。
- 服务端更换模型或输入尺寸后，缓存键会变化，客户端不需要处理旧缓存失效。
- `dataUrl` 可能较大，客户端如需长期保存，建议转为 Blob 或单独缓存。
- 深度图是归一化灰度 PNG，颜色越亮或越暗只表示相对深度，不表示跨图片可比较的绝对距离。
